# Collaboration Workflow (GitHub)

Use the Feature Branch workflow to maintain integrity:

1. Branching: Create feature/analysis-name for every new ticket.
2. Commits: Atomic changes with clear messages (e.g., fix: update logic for active users).
3. Pull Requests (PR): Mandatory peer review to catch logic errors before merging to main.
4. Environment: Always include a requirements.txt to ensure everyone runs the same library versions.
